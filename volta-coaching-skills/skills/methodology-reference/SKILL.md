# Methodology Reference

Instant lookup for any element of Volta's coaching methodology. Return concise, actionable answers.

## Quick Lookups

### Problem Hypothesis Template
When asked about the problem template or problem statement:

"We believe that [WHO] experiences [NEGATIVE IMPACT] when [TRIGGER] happens because of [ROOT CAUSE]. Alleviating that pain would allow them to achieve [FUTURE STATE]."

Components: Who (specific person/role) → Trigger (moment of awareness) → Problem (customer's words) → Impact (quantifiable) → Root Cause (deeper than symptom) → Future State (customer's world after)

### Risk Framework
When asked about risks:

4 risks scored 1-10, tracked continuously:
- **Value Risk:** Will customers pay? (Evidence: interviews, revenue, LOIs)
- **Feasibility Risk:** Can the team build it? (Evidence: shipping cadence, team composition)
- **Usability Risk:** Will users adopt it? (Evidence: retention, engagement, NPS)
- **Viability Risk:** Can it become a business? (Evidence: MRR, burn, pricing, market size)

Risk by stage: Value Baseline = Value Risk | First Customer = Viability | $10K = Feasibility + Viability | $100K = Viability + Usability | $1M = All low

### Milestones
When asked about stages or milestones:

1. **Value Baseline** — Pre-revenue, 5+ discovery interviews, problem validated
2. **First Paying Customer** — First $, demo conversion, repeatable path emerging
3. **X ($10K ARR)** — Outbound activity, pipeline metrics, growth engine forming
4. **10X ($100K ARR)** — 20%+ MoM sustained, team beyond founders, market position clear
5. **$1M ARR** — Graduation-ready, institutional readiness

### Accountability Standards
When asked about commitments or accountability:

- **20% MoM growth** as minimum evidence floor (post-first-customer)
- **Early stage equivalent:** 5 discovery transcripts / 2 weeks + 5 demos / 2 weeks
- **Bidirectional commitments:** Every session produces founder AND coach commitments
- **Escalation:** Support → Tighter metrics → Viability discussion → Compassionate off-ramp
- **Off-ramp ≠ exit:** Community access continues. Door stays open.

### Experiment Standard
When asked about experiments:

Every experiment needs 4 things:
1. **Hypothesis:** "We believe [action] will result in [outcome] for [audience]"
2. **Metric:** What number tells us it worked?
3. **Timeline:** By when? (typically 2-4 weeks)
4. **Decision Threshold:** "If we don't see [X], we pivot"

### Red Flags
When asked about warning signs:

8 observable flags:
1. Can't describe problem without describing product → more discovery needed
2. "Strong pipeline" but nothing closes → probe exact conversion numbers
3. Doing everything themselves → "who does sales full-time?"
4. Pilot keeps stalling → problem may not be urgent enough
5. "We serve everyone" → force single beachhead choice
6. Team issues recurring → probe commitment and structure
7. Founder seems off → address wellness directly
8. Enterprise target with 2-person team → redirect to faster-moving segments
